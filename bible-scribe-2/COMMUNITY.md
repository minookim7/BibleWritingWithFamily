# 함께 필사하기 — 커뮤니티 기능 설계

가족·소그룹(~10명)이 **서로의 필사 진도를 함께 보는** 기능을 어떻게
구축하는지 단계별로 정리합니다. 지금 앱은 로컬 저장이라 각자 기기에만
기록이 남는데, 이걸 온라인으로 공유하려면 "모두가 같은 곳에 기록을
저장하고, 서로의 기록을 읽을 수 있게" 만들면 됩니다.

---

## 큰 그림

```
[각자의 브라우저]                    [공용 데이터베이스]
  성경 필사 앱  ──진도 저장──▶   Supabase
      ▲                              │
      └──────다른 사람 진도 읽기──────┘
```

핵심은 세 가지입니다.
1. **로그인** — 누가 필사했는지 구분해야 하니 사람마다 계정이 필요합니다.
2. **공용 저장소** — 진도를 각자 브라우저가 아니라 온라인 DB에 저장합니다.
3. **그룹** — "우리 가족" 같은 묶음을 만들고, 그 안의 사람들끼리만 진도를 봅니다.

규모가 ~10명이라 무료 티어(Supabase)로 비용 없이 충분합니다.

---

## 왜 Supabase인가

- 무료 티어로 소그룹은 넉넉합니다 (월 5만 명 인증, 500MB DB).
- 로그인(이메일 매직링크), 데이터베이스, 권한 관리가 한 곳에 있습니다.
- 이 계정에 이미 Supabase 연결이 있어 바로 시작할 수 있습니다.
- **Row Level Security(RLS)**로 "같은 그룹 사람만 서로 본다"를 DB 차원에서
  강제할 수 있어, 프라이버시가 안전합니다.

---

## 데이터베이스 구조 (테이블 4개)

```
profiles            사용자 프로필
  id (uuid)         ← 로그인 계정과 1:1
  display_name      화면에 보일 이름 (예: "엄마", "지훈")
  created_at

groups              그룹 (예: "우리 가족")
  id
  name
  invite_code       초대 코드 (이걸 아는 사람만 가입)
  created_by

group_members       그룹 ↔ 사람 연결
  group_id
  user_id
  joined_at

sessions            완료한 필사 기록 (지금 로컬에 쌓이는 것과 동일)
  id
  user_id
  book_id, chapter
  chars, ms, cpm, accuracy, mistakes
  completed_at
```

지금 앱의 `recordSession()`이 로컬에 저장하는 구조를 그대로 `sessions`
테이블로 옮기는 것이라, 데이터 모양이 거의 같습니다. 이전이 자연스럽습니다.

---

## 프라이버시 규칙 (RLS)

Supabase의 Row Level Security로 다음을 DB가 강제하게 합니다.

- 내 세션은 내가 읽고 쓸 수 있다.
- **같은 그룹에 속한 사람의 세션만** 읽을 수 있다. (남의 그룹은 못 봄)
- 그룹 가입은 초대 코드가 있어야 한다.

이렇게 하면 앱 코드가 실수해도 데이터베이스 단에서 막히므로,
가족 기록이 외부에 새지 않습니다.

---

## 단계별 구축 순서

### 1단계 — 클라우드 저장으로 전환 (혼자서도 유용)
먼저 커뮤니티 없이, 진도를 Supabase에 저장하는 것부터 합니다.
- Supabase 프로젝트 생성 → 위 `sessions` 테이블 만들기
- 앱에 Supabase 클라이언트 추가
- `recordSession()`이 로컬 대신(또는 로컬과 함께) Supabase에 쓰도록 수정
- 이 단계만 해도 **기기 간 이어하기**(Mac↔폰)가 됩니다

### 2단계 — 로그인 추가
- Supabase Auth의 **이메일 매직링크** 사용 (비밀번호 불필요)
- 이메일 입력 → 메일로 온 링크 클릭 → 로그인 완료
- 로그인한 사람의 `user_id`를 세션에 붙여 저장

### 3단계 — 그룹 만들기 / 초대
- "그룹 만들기" → 이름 입력 → 초대 코드 자동 생성
- 초대 코드를 가족에게 공유 (카톡 등)
- 코드 입력 → 그룹 가입

### 4단계 — "함께 보기" 화면
- 같은 그룹 멤버들의 최근 진도를 한 화면에:
  ```
  우리 가족                    이번 주
  ─────────────────────────────────
  엄마    창세기 12장까지    3,200자   ●●●●○
  지훈    시편 23편          1,100자   ●●○○○
  나      요한복음 5장       2,400자   ●●●○○
  ```
- 순위·경쟁보다 "함께 걷는 느낌"에 맞춰 잔잔하게 (리추얼 정신 유지)

---

## 주의할 점

- **경쟁 프레이밍 피하기**: 이 앱은 리추얼·명상 도구입니다. 등수·1등 강조보다
  "누가 어디를 지나고 있는지 함께 본다" 정도의 부드러운 표현이 어울립니다.
- **탈퇴·삭제**: 가족이 자기 기록을 지울 수 있어야 합니다 (GDPR/상식 차원).
- **오프라인 대비**: 온라인 저장으로 가면 인터넷이 없을 때를 대비해,
  로컬에 임시 저장 후 접속되면 동기화하는 방식이 안전합니다.
- **비용**: ~10명은 무료 티어 안입니다. 수백 명 이상이면 유료 검토.

---

## 다음에 할 일

이 문서는 설계까지입니다. 실제 구축을 원하시면:
1. Supabase 프로젝트를 만들고 (또는 기존 연결 사용)
2. 위 1단계(클라우드 저장)부터 함께 코드로 옮기면 됩니다.

1단계만 해도 "기기 간 이어하기"라는 큰 편의가 생기니, 커뮤니티 이전에
거기부터 시작하는 걸 권합니다.
