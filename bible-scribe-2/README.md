# 성경 필사 2.0 — 리추얼 필사

typing.works의 "Transcribing as Ritual" 정신을 담은 성경 필사 웹앱.
속도 경쟁이 아니라, 한 구절씩 차분히 옮겨 적으며 머무는 도구입니다.

- **개역한글판 66권 전체** (구약 39 + 신약 27, 30,960절)
- **고정 시선(fixed-gaze)** 필사 화면 — 현재 구절이 화면 중앙에 머뭅니다
- **절 단위 표시** — 각 절 앞에 절 번호 배지
- **자동 저장 · 이어하기** — 앱을 껐다 켜도 그 지점부터 (로컬 저장)
- **세 가지 테마** — 먹(어둠) / 한지(밝음) / 세피아
- 실시간 통계(타/분·정확도·진행률·시간), 기록·달력·CSV 내보내기
- 검색, 즐겨찾기, 암송 모드, 배경음
- **서비스워커 없음** — 오프라인 관련 문제 없이 웹에서 바로 동작

## 로컬에서 실행

정적 사이트라 아무 웹서버로나 열립니다. (`file://` 직접 열기는 안 됨)

```bash
cd bible-scribe-2
python3 -m http.server 8000
# → http://localhost:8000
```

## Vercel 배포 (권장)

이 앱의 목표는 "웹에 올려서 어디서든 접속"입니다. Vercel이 가장 간단합니다.

### 방법 A — 웹 대시보드 (제일 쉬움)
1. 이 폴더를 GitHub 저장소에 올립니다.
2. vercel.com → Add New → Project → 그 저장소 선택.
3. 프레임워크 프리셋은 **Other**(정적) 그대로 두고 Deploy.
4. 몇 초 뒤 `https://<프로젝트>.vercel.app` 주소가 나옵니다. 끝.

### 방법 B — CLI
```bash
npm i -g vercel
cd bible-scribe-2
vercel        # 안내에 따라 로그인 후 배포
vercel --prod # 프로덕션 주소로 승격
```

배포되면 그 주소를 Mac·폰 어디서든 열 수 있고, 홈 화면에 추가하면
앱처럼 쓸 수 있습니다. 진행 상황은 각 기기 브라우저에 저장됩니다.

## 진행 상황 저장 방식 (로컬)

진행 상황은 브라우저 LocalStorage에 `progress:<책>:<장>` 키로 저장됩니다.
- 자동 저장은 입력이 멈출 때(0.5초)와 앱을 닫을 때 이뤄집니다.
- 한 장을 완료하면 그 진행분은 지워지고, 기록(records)에 세션이 남습니다.
- **주의**: 로컬 저장이라 기기마다 따로입니다. Mac에서 하던 걸 폰에서
  이어받으려면 아래 "클라우드/커뮤니티 확장"이 필요합니다.

## 클라우드 · 커뮤니티로 확장하려면 (나중에)

가족·소그룹(~10명)이 서로의 진도를 보는 기능은 아래 순서로 얹을 수 있습니다.
자세한 설계는 `COMMUNITY.md`를 참고하세요.

1. **Supabase 프로젝트 생성** (무료 티어로 충분)
2. 진행/세션 데이터를 LocalStorage → Supabase 테이블로 이동
3. 간단한 로그인(이메일 매직링크) 추가
4. 그룹 테이블 + 멤버들의 최근 진도를 보여주는 "함께 보기" 화면
